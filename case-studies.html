01 FLAGSHIP

Mr. George — the PM in my pocket.

Internally codenamed Mr. George. Seventeen AI agents that read my emails, write my submittals, search 200+ drawings, dispatch to a research panel, and page me on Discord when something breaks. The const-agent web UI is the cockpit.

Problem

A project engineer on a $2B job spends most of their day shuffling: copying RFI responses into trackers, searching drawings for one detail, drafting submittal cover letters, chasing down inspector confirmations. LLMs are good at every one of these tasks individually. Nobody had wired them together for construction PM yet.

Approach

Built a multi-channel agent framework in Python. The same 17 agents — submittal coordinator, RFI triager, schedule analyst, drawing indexer, takeoff specialist, tool tracker, foreman roster, advisory panel of five — answer in Discord, Teams, and a dedicated web UI. Each channel routes to the right agent based on the message.

Model providers are hot-swappable: Anthropic (Claude Opus), OpenAI (Codex via ChatGPT OAuth), MiniMax M2.7, Gemini, Ollama for local. If one goes down, the next provider picks up. Vision is enabled — paste an image directly into Alfred and he'll read it.

  • Dispatch agent analyzes intent, routes to one of 17 specialists
  • Advisory panel: 5 research agents + a manager, all with Drive folder routing
  • Multi-vault routing — personal brain (192 notes) + project brain (25 LACC notes)
  • Sheets create / append / format from chat (Submittal, RFI, Tools, Embed trackers)
  • Drawing indexer runs fitz + pdfplumber + Gemini over 200+ sheet PDFs
  • Voice channel via ElevenLabs (Daniel / Charlie) + Web Speech API mic
  • Tool tracker assets-sync UI shipped today (4/29) — drop a ToolHound xlsx, click Sync
  • macOS Keychain for all provider keys — zero secrets in git
Every node in the brain is a markdown file. Every connection is a wikilink. Alfred reads them, links them, and surfaces the right one when you ask. The user authors the brain by editing .md files in Obsidian — the AI is just the librarian.
Result

Cuts a typical RFI/submittal prep from ~40 minutes to ~6. Drawing-search questions that used to take 10 minutes of sheet-flipping get answered in seconds. The Tools tab now ingests a ToolHound asset report and reconciles to the Assets sheet in one click. The full audit trail lives in SQLite — every question, every response, every tool call.

02 LIVE

Command Center — every Webcor sheet in one tab.

Single-pane-of-glass for the LACC PE team. Embeds, tools, ACP coordination, submittals, RFIs — every spreadsheet the team tracks, surfaced in one Wangan-OS dashboard.

Problem

The LACC South Hall PE team runs on five different Google Sheets across four tabs apiece. Status is scattered — to know "are we behind on submittals" you open a sheet, scroll, eyeball, repeat for the next thing. Nobody had a default landing page.

Approach

Static GitHub Pages SPA. Reads live from every team sheet via Apps Script proxies. Carbon-fiber + cyan neon styling, Chart.js dashboards, drill-down per category. Mobile-friendly so foremen can pull it up in the field.

  • Embeds — full LACC Embed Tracker (Detailed Tracker)
  • Tools — Assets, Transfers, Foremen roster aggregated
  • ACP Coordination — Vertical Bar (Table B Locations, Pile Types)
  • Submittals & RFIs — RFI Log, Gap Analysis surfaced
  • Tachometer glow + carbon-fiber theme — Wangan OS visual language
Result

Featured app on the OpenYap launcher. Replaces the morning "open five tabs" ritual with one URL. Status is at-a-glance — and the rest of the team can see it on their phone when they're not at a desk.

03 LIVE

CUP Dashboard — the job site in your pocket.

Real-time concrete-pour platform for the LACC project. Pour plans, embed verification, live trucks, mix designs, inspection ticker, and a camera-driven bulk PDF scanner — all synced across every device in the field.

Problem

On a pour day you have 50+ moving parts: concrete trucks arriving, rebar placement status, embeds to verify, inspectors to coordinate, and a PM chewing through a printed footing map. The existing tools were: a spreadsheet, a clipboard, and walkie-talkies.

Approach

Static GitHub Pages SPA. Pour ticket indexing now uses Drive as the source of truth (we dropped the Firebase write path for tickets — fewer moving parts, audit trail in the sheet). Foremen drag footing shapes on the map, long-press embeds to step through a checklist, tap a truck arriving to log it live, and a new bulk scanner uses the camera + edge detection to ingest multi-page ticket PDFs.

  • Footing map with drag-edit + SVG status dots (green / amber / red-pulse)
  • Live Pour tab — donut chart, truck log, inspection ticker (?view=live)
  • Embed tracker with 10 CUP types (51A–90A) + long-press checklist popup
  • Mix design panel — PSI, aggregate, slump, LC3 cement
  • Bulk scanner — live camera feed + edge detection for multi-page PDF ingest
  • Apps Script → Sheet write-back for auditability
  • Alfred chat embedded with embed + mix-design context injection
Result

Shipped live to the team. Three sections of pour tracked end-to-end. Inspectors are using it to confirm embed placement without a site visit. Shared with superintendent + project executive. Theme overhaul matched the roadmap site so OpenYap reads as one product family.

04 LIVE

Formwork — budget intelligence that understands concrete.

OpenYap's own budget tool. Categories are formwork bays. Planned spend pours in as concrete. Line items are studs in the form. Sankey flow, Cmd-K palette, health gauge, 3D lattice front and center.

Problem

Running a venture means tracking every dollar of the $1.2M seed across LLC / legal / infra / dev / marketing / ops. Spreadsheets work but don't let you think in line items inside each budget, and they don't show the flow of money. I wanted a construction-native UX for money.

Approach

Firebase-backed single-file app. Each category drills into line items with statuses (planned / committed / done). The Plan view toggles between a Sankey flow and a Grid; scenarios let you fork the budget for what-ifs; Cmd-K palette hops to anything; a health gauge surfaces variance against plan in one glance. The Nodes view is a 3D formwork lattice — wireframe bays sized by planned total, concrete fill ∝ actual spend.

  • Sankey Plan view + Flow / Grid mode toggle
  • Scenarios — fork the budget, compare side-by-side
  • Cmd-K command palette to jump anywhere
  • Health gauge — variance against plan, at a glance
  • 3D formwork lattice (Compare view) and Excel export for compliance
Result

Real-time sync across devices. Renamed from "Lumino" to "Formwork" — the tool's metaphor now matches the ground it's standing on. The board can see budget plans flow as if they were concrete.

05 LIVE

OpenYap Roadmap — the investor deck as a building.

A scroll-driven 3D construction site. Each section builds a floor. The crane climbs. Nodes orbit. It's a pitch deck that moves — and we ship two themes for two audiences.

Problem

A 5-phase business plan shown on a slide deck is boring. A 5-phase business plan where the building literally rises as you scroll is memorable. The goal was turning the OpenYap thesis into a 90-second experience.

Approach

Three.js scene locked behind a WebGL shader background. 45 floors of glass assemble as you scroll, a lattice crane climbs with them, 800 particles drift, 13 material stacks tilt and shadow. GPU is auto-detected so the experience scales from a phone to an M3 Max without dropping frames.

  • Scroll position → camera & assembly progress
  • Aurora shader mesh gradient background
  • Floating node network (80 nodes, orbital connections)
  • Glassmorphic agreement panels with floor-highlight on hover
  • GPU auto-detection + aggressive perf tuning
  • Stitch variant — alternate theme for cleaner-skin audiences
Result

Shown to the Webcor board. The scroll-to-build animation hit the room hard — feedback was "this feels like the pitch." Hero stats now: $1.2M seed, 1,000%+ ROI, $432M+ Y5 TAM. Two themes deployed (default + Stitch) so we can show whichever fits the room.
