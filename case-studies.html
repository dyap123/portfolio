01 CORE

Const-Agent — the PM in my pocket.

A 15-agent AI system that reads my emails, writes my submittals, searches 200+ drawings, and pages me on Discord when something breaks.

Problem

A project engineer on a $2B job spends most of their day shuffling: copying RFI responses into trackers, searching drawings for one detail, drafting submittal cover letters, chasing down inspector confirmations. LLMs are good at every one of these tasks individually. Nobody had wired them together for construction PM yet.

Approach

Built a multi-channel agent framework in Python. The same 15 agents — submittal coordinator, RFI triager, schedule analyst, drawing indexer, takeoff specialist, advisory panel — answer in Discord, Teams, and a dedicated web UI. Each channel routes to the right agent based on the message.

Model providers are hot-swappable: Anthropic (Claude Opus), OpenAI (Codex via ChatGPT OAuth), MiniMax, Gemini, Ollama for local. If one goes down, the next provider picks up.

  • Dispatch agent analyzes intent, routes to one of 14 specialists
  • Drawing indexer runs pdfplumber + Gemini over 200+ sheet PDFs
  • Voice channel via ElevenLabs + Whisper (Alfred)
  • macOS Keychain for all provider keys — zero secrets in git
Result

Cuts a typical RFI/submittal prep from ~40 minutes to ~6. Drawing-search questions that used to take 10 minutes of sheet-flipping get answered in seconds. The full audit trail lives in SQLite — every question, every response, every tool call.

02 LIVE

CUP Dashboard — the job site in your pocket.

A real-time concrete-pour dashboard for the LACC project. Pour plans, embed tracking, live trucks, inspection ticker — all synced across every device in the field.

Problem

On a pour day you have 50+ moving parts: concrete trucks arriving, rebar placement status, embeds to verify, inspectors to coordinate, and a PM chewing through a printed footing map. The existing tools were: a spreadsheet, a clipboard, and walkie-talkies.

Approach

Static GitHub Pages SPA + Firebase Realtime DB. Everyone in the field sees the same state. Foremen drag footing shapes on the map, long-press embeds to step through a checklist, and tap a truck arriving to log it live.

  • Footing map with drag-edit + SVG status dots (green / amber / red-pulse)
  • Live Pour tab — donut chart, truck log, inspection ticker
  • Embed tracker with 10 CUP embed types (51A – 90A) + long-press checklist
  • Apps Script → Sheet write-back for auditability
  • Alfred chat embedded in Brain tab — ask the dashboard questions
Result

Shipped live to the team. Three sections of pour tracked end-to-end. Inspectors are using it to confirm embed placement without a site visit. Shared with superintendent + project executive.

03 LIVE

OpenYap Roadmap — the investor deck as a building.

A scroll-driven 3D construction site. Each section builds a floor. The crane climbs. Nodes orbit. It's a pitch deck that moves.

Problem

A 5-phase business plan shown on a slide deck is boring. A 5-phase business plan where the building literally rises as you scroll is memorable. The goal was turning the OpenYap thesis into a 90-second experience.

Approach

Three.js scene locked behind a WebGL shader background. 45 floors of glass assemble as you scroll, a lattice crane climbs with them, 800 particles drift, 13 material stacks tilt and shadow.

  • Scroll position → camera & assembly progress
  • Aurora shader mesh gradient background
  • Floating node network (80 nodes, orbital connections)
  • Glassmorphic agreement panels with floor-highlight on hover
Result

Shown to the Webcor board. The scroll-to-build animation hit the room hard — feedback was "this feels like the pitch."

04 LOCAL

Vault Brain — my second brain, visualized.

A 3D neural network rendered from my Obsidian vault. Every note is a light. Every wikilink is an edge. Alfred sits in the middle, voice-enabled.

Problem

Obsidian's built-in graph view is flat, small, and doesn't feel like knowledge. I wanted something that felt like a mind — glowing, alive, interrogable.

Approach

FastAPI proxy reads the vault, parses wikilinks, serves node + edge JSON. Three.js renders sprite-based lights with additive blending (no bloom — cleaner). Boot animation builds the brain node-by-node. Alfred chat streams responses through Codex/Claude/MiniMax; persistent SQLite chat history.

Result

192-note personal vault and 25-note project vault both render. ENHANCE mode boosts importance-based sizing. Used daily as the control surface for my second brain.

05 LIVE

Formwork — budget intelligence that understands concrete.

OpenYap's own budget tool. Categories are formwork bays. Planned spend pours in as concrete. Line items are studs in the form. Three.js lattice front and center.

Problem

Running a venture means tracking every dollar of the $1.2M seed across LLC / legal / infra / dev / marketing / ops. Spreadsheets work but don't let you think in line items inside each budget. I wanted a construction-native UX for money.

Approach

Firebase-backed single-file app. Each category has drill-in line items with statuses (planned / committed / done). New Compare view: stacked-area plan flow, side-by-side category variance, flat cross-category line-item list. Nodes view is a 3D formwork lattice — wireframe bays sized by planned total, concrete fill ∝ actual spend.

Result

Ships real-time sync across my devices. Excel export for compliance. Renamed from "Lumino" to "Formwork" — now the tool's metaphor matches the ground it's standing on.
